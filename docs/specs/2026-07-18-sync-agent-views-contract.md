# Contract — `scripts/sync-agent-views.mjs`

> **Status:** ACTIVE (amended after staff review — native memory agent-owned; marked+no-manifest adopts).
> **Fleet model:** Option A — plugin `rules/` is fleet-canonical; see
> `docs/decisions/2026-07-18-fleet-rules-canonical.md`. v1 CLI is per-`--root`; fleet iteration
> (`--all` / Studio) is the next sync increment before UI hardens on per-project-only editing.

## 1. Purpose

One canonical markdown vault (git-tracked) generates **one-way** agent views for rules,
plus an MCP memory feed. Clobbering hand-edits of generated rule files is a hard failure.

## 2. Canonical sources (git-tracked)

| Path | Role |
|------|------|
| **Plugin** `rules/*.md` | **Fleet** shared rule bodies (Option A) |
| `<target>/rules/*.md` | v1 per-root source when syncing a single `--root` (dogfood/CI); fleet mode sources plugin |
| `<target>/memory/*.md` | Project memory notes (`type: memory` preferred) → **MCP feed + Studio only** |

## 3. Generated targets (v1)

| Target | From | Notes |
|--------|------|-------|
| `<project>/.claude/rules/<name>.md` | `rules/<name>.md` | HTML `luna:generated` comment only (no visible `# GENERATED` heading — context noise) |
| `<project>/.cursor/rules/<name>.mdc` | `rules/<name>.md` | Cursor FM + generated banner |
| `docs/generated/mcp-memory-feed.json` | `memory/*.md` | Stable JSON: `_luna_generated`, `schema_version: 1`, no volatile timestamps |

### Explicitly NOT generated (v1)

| Path | Why |
|------|-----|
| `~/.claude/projects/<slug>/memory/*` | **Agent-owned.** Bidirectional; agents write here during sessions. Injecting/replacing would fight the agent and wipe index pointers. |
| `MEMORY.md` (native) | Same — agent-authored index. |

Canonical `memory/*.md` is consumed via MCP feed / Studio. Native-memory merge (namespaced subdir + merge-not-replace index) is deferred to Phase 3.

## 4. Never write (hard skip)

- `lessons.md` / `lessons.mdc`
- `RULES.md` / `RULES.mdc`
- `*.local.md` / `*.local.mdc`
- `luna.mdc`

## 5. Markers

**Claude rules** — HTML comment only:

```markdown
<!-- luna:generated by scripts/sync-agent-views.mjs — DO NOT EDIT; edit rules/<name>.md -->
```

**Cursor rules** — comment + visible `# GENERATED` heading after YAML front-matter.

**MCP feed** — `"_luna_generated": "scripts/sync-agent-views.mjs"` + `schema_version: 1`.

Detection: file is "ours" iff it contains `luna:generated by scripts/sync-agent-views.mjs`
(or the JSON `_luna_generated` field).

## 6. No-clobber / hash guard

Manifest (gitignored): `.luna/agent-views-manifest.json`

| Existing target | Condition | Action |
|-----------------|-----------|--------|
| missing | — | write `desired`, record hash |
| present | `current === desired` | no-op |
| present | marked **and** no manifest entry | **adopt + regenerate** (fresh clone / CI); warn once; record hash |
| present | marked **and** `sha256(current) === manifest[path].sha256` | overwrite with `desired`, update manifest |
| present | marked **and** manifest hash mismatch | **ABORT** exit 2 + diff (genuine local edit) |
| present | no marker | **ABORT** exit 2 (hand-authored) |

`--force` is **not** a clobber bypass.

## 7. Orphans

If a canonical source is deleted but a previously generated file remains, sync **warns**
(lists orphans from the manifest) and drops the manifest entry. v1 does **not** auto-delete
the orphan file.

## 8. Idempotence

Two successive successful syncs: second is a full no-op; `--check` green twice.

## 9. CLI

```bash
node scripts/sync-agent-views.mjs [--root <path>] [--check] [--dry-run]
```

## 10. Out of scope (v1)

- Writing into native Claude memory
- User-level `~/.claude/CLAUDE.md` / `~/.cursor/rules/*`
- UI diff-preview (Task 3)
- Auto-deleting orphan generated files

## 11. Migration (one-time)

Unmarked `.claude/rules/*.md` / `.cursor/rules/*.mdc` abort sync. Move them aside
(keep `lessons.*`), run sync, diff-compare. In-repo dogfood migration is part of Task 1 acceptance.
