# Knowledge stack

**Read order (L1–L4):** invoke **`vibe-rules`** §0 or read plugin `docs/VIBE_RULES.md` §0. One question → one canonical layer.

## Doc authoring rules

- Project docs contain invariants, diagrams, contracts (shapes only) — never implementation.
- Use `[ref]` tags in prose; resolve all paths in a `## File index` table at the doc bottom.
- Never list callers/callees in markdown — GitNexus owns that graph.
- One concern per file; ownership table lives in `docs/README.md`.
- Archive (`docs/_archive/`) is git-history only — do not link from living docs.
- Docs >300 lines: warn. Docs >500 lines: act — run `doc-simplify` + File index pass.

## Catalog (`docs/README.md`)

Every project should have `docs/README.md`. Required sections:

- **Doc map** — mermaid flowchart of doc dependencies (entry → domain docs)
- **Catalog table** — `File | Role | Read when | Agent keywords` (regenerated by `build-docs-index.mjs` between `luna:generated:catalog` markers)
- **Ownership rules** — `Topic | Canonical doc | Never duplicate in`
- **Read order** — numbered list, ≤4 steps

`AGENTS.md` links to `docs/README.md`; it does not duplicate the catalog.
Archive (`_archive/`) excluded from catalog — git history only.

## Lifecycle + front-matter

Unified schema: `templates/docs/FRONTMATTER.md`. Buckets: `docs/pre-official/`, OFFICIAL paths,
`docs/post-official/`, `docs/decisions/`.

## llms.txt

Generated by `scripts/build-docs-index.mjs` at repo root. Points to `docs/README.md` and `AGENTS.md`. ≤20 lines.

## Multi-repo

Submodules with independent `.gitnexus/` indexes: run `gitnexus-group-setup` skill to wire
`group_sync` + `group_query` — one call searches across all repos in the group.
